# Linux Permission

Linux는 파일과 디렉토리에 대한 접근 권한을 관리한다.

권한은 다음 세 사용자에게 적용된다.

-   Owner : 파일 소유자
-   Group : 소유 그룹
-   Others : 그 외 사용자

------------------------------------------------------------------------

## 권한 종류

  권한   의미
  ------ ---------------
  r      읽기(Read)
  w      쓰기(Write)
  x      실행(Execute)

------------------------------------------------------------------------

# chmod

파일 또는 디렉토리의 권한을 변경하는 명령어이다.

## 사용법

``` bash
chmod 권한 파일명
```

### 숫자 방식

    숫자 권한
  ------ ------
       7 rwx
       6 rw-
       5 r-x
       4 r--
       3 -wx
       2 -w-
       1 --x
       0 ---

예시

``` bash
chmod 764 hello
```

### 문자 방식

권한을 추가하거나 제거할 수도 있다.

``` bash
chmod g+x hello
chmod o-r hello
chmod u+w hello
```

------------------------------------------------------------------------

# chown

파일의 소유자를 변경하는 명령어이다.

## 사용법

``` bash
sudo chown 사용자 파일명
```

예시

``` bash
sudo chown root hello
```

------------------------------------------------------------------------

# chgrp

파일의 소유 그룹을 변경하는 명령어이다.

## 사용법

``` bash
chgrp 그룹명 파일명
```

예시

``` bash
chgrp users hello
```

------------------------------------------------------------------------

# Special Permission

## SetUID

실행 파일이 실행될 때 파일 소유자의 권한으로 실행된다.

예시

``` text
-rwsr-xr-x
```

------------------------------------------------------------------------

## SetGID

실행 파일이 실행될 때 소유 그룹의 권한으로 실행된다.

------------------------------------------------------------------------

## Sticky Bit

공용 디렉토리에서 파일 소유자와 root만 파일을 삭제할 수 있도록 하는
권한이다.

예시

``` text
drwxrwxrwt
```

대표적인 디렉토리

``` text
/tmp
```

------------------------------------------------------------------------

# 핵심 정리

-   Linux는 Owner, Group, Others로 권한을 관리한다.
-   권한은 읽기(r), 쓰기(w), 실행(x)로 구성된다.
-   `chmod`는 파일 권한을 변경한다.
-   `chown`은 파일 소유자를 변경한다.
-   `chgrp`는 파일의 소유 그룹을 변경한다.
-   특수 권한에는 SetUID, SetGID, Sticky Bit가 있다.
